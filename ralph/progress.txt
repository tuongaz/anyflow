## Codebase Patterns
- apps/viewer uses @xyflow/react for React Flow canvas rendering
- convertNode/convertConnector are the bridge from Demo types to React Flow Node/Edge types
- colorTokenStyle from lib/color-tokens.ts handles color token → CSS style conversion
- STYLE_BY_KIND and STYLE_BY_STYLE in demo-to-flow.ts control connector dash patterns
- viewer-api.ts in apps/viewer/src/lib/ is the single source for all API calls (fetchFlow, fetchFlows) pointing to https://seeflow.dev/api
- exportToCloud in use-export-to-cloud.ts signature: (projectId, email, name, visibility: 'public'|'link', previewDataUrl?); Visibility type is exported; zip key is seeflow.json (public) or seeflow.private.json (link)
- When adding required params to a hook used by a component that will be updated in a later story, update the component with a placeholder value to keep typecheck passing
- FlowCard uses inline styles (not CSS classes) consistent with other viewer components; aspectRatio '16 / 9' for the canvas preview container
- relativeDate helper: 0d=today, 1d=yesterday, <30d=Nd ago, <12mo=Nmo ago, else Ny ago
